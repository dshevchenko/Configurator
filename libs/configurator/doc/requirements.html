Necessary Boost libraries

Boost.Configurator uses only ISO C++ code and following Boost libraries:
  • Spirit
  • Fusion
  • Range
  • Filesystem
  • Pointer Container
  • Asio
  • System
  • Bind
  • Function
  • Foreach
  • Assign
  • Utility
  • Thread
  • String Algo
  • Member Function
  • Lexical Cast
  • Regex
    •
Note that Filesystem, Thread, Regex and System (required for Asio) libraries are auto linking, so you must build them and link with your program.

But in accordance with the principle "you only pay for what you use", System and Regex libraries required only when you use semantics check, and Thread library required only when you use Singleton-variant of core class.

Compiling programs with Boost.Configurator

Simplest example of compiling program with Boost.Configurator (in Linux): 
# g++ -o my_test main.cpp -lboost_filesystem
If you want use Singleton-variant of core class, you must define WITH_SINGLETON flag and link Boost.Thread library: 
# g++ -o my_test main.cpp -DWITH_SINGLETON -lboost_filesystem -lboost_thread
If you want use semantics check, you must define WITH_SEMANTIC_CHECK flag and link Boost.System and Boost.Regex libraries: 
# g++ -o my_test main.cpp -DWITH_SEMANTIC_CHECK -lboost_filesystem -lboost_system -lboost_regex
Note that in stable release Boost.Regex will not be used, Boost.Spirit will replace it.
